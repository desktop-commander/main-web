What this prompt does

Get PostHog analytics running with custom event tracking — AI installs the SDK, configures autocapture, sets up custom events for your key user actions, and optionally adds a reverse proxy for better ad-blocker resistance.

How it works

  1. 1.Run this prompt in Desktop Commander with your project path
  2. 2.AI adds PostHog SDK to your project configuration
  3. 3.Sets up autocapture and defines custom events
  4. 4.Configures privacy settings and optional reverse proxy

# TASK NAME
Setting up Posthog Analytics with custom events

## Automation Mission Statement
You are an expert in web analytics and PostHog implementation. Your role is to guide the user through streamlined, best-practice setup and tracking using the capabilities of Desktop Commander.

---

## Multi-Chat Workflow Task Type
This workflow may span multiple chats to avoid context limits. Progress is tracked in the `posthog-setup-progress.md` file after each phase, containing:
- Instructions, context, requirements, findings, and next steps.
- Recap of phases, outputs, and current status.
- Next tasks, pending items, and critical file locations.

**Continuation:**  
Start a new chat session when the conversation is too long, you want to focus on a different aspect, or are returning to work after a break. In a new chat:  
"Continue task type - please read posthog-setup-progress.md to understand where we left off, then proceed with the next phase."

---

## Streamlined Phase-Based Methodology (MAX 3 PHASES)
- **Phase 1 - Foundation:** Core tool installation and initial capture setup.
- **Phase 2 - Key Implementation:** Integration, event tracking, custom events/properties, and privacy compliance.
- **Phase 3 - Finalization:** Optimization, updates, QA, and documentation.

After each phase: progress is updated and confirmation requested before advancing.

---

## Desktop Commander Integration

Use Desktop Commander where needed to complete this task.

Answering the following questions will help customize the setup. Skip or use defaults if preferred.

**Essential Setup Questions (Optional):**
1. Where is your website hosted? (e.g., Vercel, Netlify, AWS, custom server)  
   _Determines appropriate snippet and reverse proxy approach_  
2. Which key user events do you want to track? (e.g., sign up, purchase, button click, page view)  
   _Defines event schema for custom analytics_  
3. Any custom user properties or tracking needs?  
   _Ensures all business-relevant data is captured_  


Quick-start options:

---

## Core Task Framework & Best Practices

- Use the official PostHog JavaScript snippet/SDK in your website's `<head>` for maximum coverage.
- Enable autocapture for pageviews, clicks, and scrolls.
- Route analytics through a reverse proxy (via managed service or your stack) to minimize tracking/ad blocker impact.
- Group related platforms (marketing site, mobile, web app) in one PostHog project for unified measurement.
- Configure and track custom events directly relevant to your business needs.
- Regularly update your PostHog version for optimal performance and security.
- Honor privacy/compliance obligations using built-in consent settings and by capturing anonymous events if required.

---

## Quality Standards

- Installation and event tracking scripts must be validated by running on target web property.
- Documentation is concise, clear, and complete for technical implementation.
- Compliance with privacy standards validated against regional/industry norms.
- File outputs must be easily readable and self-explanatory.
